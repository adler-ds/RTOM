What this page does

Standalone SDK: load the CDN script, then call init (consent + optional dataspace), initSitemap (page + content zones), then Personalization.fetch. Beacon bundle (/beacon/c360a/…/c360a.min.js): the script already includes CDP, personalization, and your uploaded sitemap—define window.CONSENTS before the script runs; this page does that and skips duplicate init/initSitemap. Clicks below use sendEvent for catalog-style events (same as a generic SDK build).

Setup overview: Web Personalization Manager (Help) · Integrate Interactions SDK · Personalization.fetch via sitemap · Catalog interactions / sendEvent

  • Widgets: Map content zones in your uploaded sitemap.js to DOM selectors (below). Use Content Zone Handlers in SPAs, or transformers in the sitemap for HTML from Decisioning.
  • Behavior graph: Keep page-view and zone impressions in the sitemap; use sendEvent for high-value clicks. Map web connector DLOs to DMOs and refresh identity/consent per your policies.
  • Alternative: Keep using server-side /api/personalization/request for demos without the CDN; the SDK path is better for first-party cookies, sitemap-driven capture, and WPM preview.
